# 🤖 AI Agent Workflow: Isolated Git Worktree

당신은 모든 작업을 수행할 때 다른 작업과의 간섭을 완벽히 차단하기 위해 반드시 고유한 Git Worktree 환경을 설계하고 사용해야 합니다.

---

## 1. 고유 작업 식별자 설계 (Naming Convention)

작업을 시작하기 전, 반드시 다음 규칙에 따라 이번 작업만을 위한 **고유 브랜치명**을 생성하십시오.

- **규칙**: `task/{YYYYMMDD}-{HHmm}-{short-description}`
- **예시**: `task/20240520-1530-fix-header-padding`
- **목적**: 기존 브랜치 및 타 작업 워크트리와의 명칭 충돌을 원천 봉쇄합니다.

## 2. 작업 공간(Worktree) 구축 규칙

- **생성 경로**: 반드시 레포지토리 루트의 `./worktrees/{고유브랜치명}` 경로를 사용합니다.
- **격리 원칙**:
  - 이미 존재하는 `./worktrees` 내의 다른 디렉토리에 절대 진입하지 않습니다.
  - 반드시 새 브랜치를 생성하며 워크트리를 추가합니다.
- **실행 명령**:
  ```bash
  git worktree add ./worktrees/{고유브랜치명} -b {고유브랜치명}
  ```

## 3. 단계별 작업 및 커밋 수행

- **이동**: 생성된 `./worktrees/{고유브랜치명}` 디렉토리 내부로 이동하여 작업을 시작합니다.
- **범위**: 사용자로부터 요청받은 "현재 작업" 외의 파일은 수정하지 않습니다.
- **커밋**: 논리적 작업 단계마다 즉시 커밋을 수행합니다. (Atomic Commit)
  - 예: `git commit -m "feat: implement logic for..."`

## 4. 작업 종료 및 환경 정리

작업이 완료되면 흔적을 남기지 않도록 다음 순서를 엄격히 준수합니다.

1. **워크트리 이탈**: 작업 디렉토리 밖(레포지토리 루트)으로 이동합니다.
2. **워크트리 제거**: `git worktree remove ./worktrees/{고유브랜치명}` 명령을 실행합니다.
3. **최종 보고**:
   - 사용한 고유 브랜치명 명시
   - 단계별 수행한 커밋 내역 요약
   - 워크트리 삭제 및 정리 완료 상태 보고

---

## ⚠️ 주의사항

- `./worktrees` 디렉토리는 작업용 임시 공간입니다. 작업 종료 후 반드시 비워져 있어야 합니다.
- 절대 메인 브랜치나 다른 활성화된 워크트리에서 직접 작업하지 마십시오.
