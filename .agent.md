## AI 에이전트가 명확한 격리 환경에서 작업을 수행하고 흔적을 남기지 않도록 하는 .agent.md 설정입니다.

## 🤖 AI 에이전트 작업 가이드라인

당신은 반드시 다음 규칙에 따라 독립된 Git Worktree 환경에서만 작업을 수행해야 합니다.

## 1. 작업 준비 (Worktree 생성)

- 모든 새로운 작업은 반드시 새 브랜치에서 진행합니다.
- 워크트리는 반드시 ./worktrees 경로 아래에 생성합니다.
- 예: ./worktrees/task-feature-name
- 현재 진행 중인 다른 워크트리로 이동하거나 간섭해서는 안 됩니다. [1]

## 2. 작업 수행 및 커밋

- 지정된 워크트리 경로 내부로 이동하여 요청된 작업만을 수행합니다.
- 단계별 커밋: 의미 있는 작업 단위마다 즉시 커밋을 수행합니다.
- 커밋 메시지는 작업 내용을 명확히 기술합니다.

## 3. 작업 종료 및 정리

- 작업이 완료되면 변경 사항을 메인 레포지토리에 푸시하거나 병합 준비를 마칩니다.
- 워크트리 제거: 작업 종료 직후 생성했던 워크트리를 반드시 삭제합니다.
- 명령어: git worktree remove <경로>
- ./worktrees 디렉토리가 비어있는지 확인합니다. [2]

## 4. 사용자 보고

- 수행한 작업 내용과 커밋 기록을 요약하여 보고합니다.
- 워크트리 생성, 작업, 삭제가 정상적으로 완료되었음을 명시합니다. [3]

---

## 🛠 실행 명령어 레퍼런스## 워크트리 생성

git worktree add ./worktrees/<branch-name> -b <branch-name>

## 워크트리 삭제

git worktree remove ./worktrees/<branch-name>
git branch -d <branch-name> # 필요 시 브랜치 삭제
